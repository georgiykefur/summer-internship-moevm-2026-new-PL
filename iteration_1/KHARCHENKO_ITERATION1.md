# Итерация 1 — Харченко Я.К. (Контроллер/Ввод)

## Что реализовано

- **`Graph`** (`src/main/java/Graph.java`) — модель графа: координаты вершин (`x[]`, `y[]`) и матрица весов `dist` (`Double.POSITIVE_INFINITY` — отсутствие ребра). Своя минимальная модель именно для этой задачи: модель Гриценко на этом этапе не подключена, части команды по-прежнему не интегрируются (см. `CLAUDE.md`).
- **`GraphParseException`** (там же) — отдельное исключение для ошибок формата файла, чтобы GUI мог показать понятное сообщение вместо голого стектрейса.
- **`GraphFileLoader`** (там же) — парсер файла графа по контракту формата (`N`, координаты вершин, матрица `N×N` с `INF`), с проверками на каждом шаге: нечисловая первая строка, неверное количество токенов в строке координат/матрицы, слишком короткий файл.
- **`StepDemoWindow`** (обновлён) — кнопка «Загрузить» теперь реально открывает файл через `JFileChooser`, парсит его `GraphFileLoader`-ом и показывает результат в новой таблице (`JTable`) с матрицей `dist`. При ошибке — сообщение в лог и диалоговое окно, приложение не падает.
- **`GraphFileLoaderTest`** — 6 unit-тестов: успешный разбор графа из спецификации, отрицательные/дробные веса, три вида некорректного файла (нечисловое `N`, неверное число токенов у вершины, неверная длина строки матрицы, слишком мало строк).

Кнопки «Шаг вперёд»/«Шаг назад» не менялись — по-прежнему листают 5 заглушечных состояний, как в итерации 0. Их подключение к настоящим шагам алгоритма — задача Гриценко на интеграции, не входит в эту часть.

## Что сознательно не сделано

- Интеграция загруженного графа с шагами алгоритма (шаги остаются на заглушках).
- Отрисовка графа на холсте (часть Кузьмина).
- Восстановление и просмотр пути, сохранение в файл — это итерация 2.

## Как собрать и запустить

```bash
javac -d out src/main/java/*.java
java -cp out StepDemoWindow
```

Проверка вручную: кнопка «Загрузить» → выбрать `data/sample_graph.txt` → в таблице снизу должна появиться матрица 4×4 с `INF` на месте отсутствующих рёбер, в логе — сообщение об успешной загрузке. Попробовать подсунуть испорченный файл (например, файл с текстом вместо числа `N`) — должно появиться диалоговое окно с ошибкой, а не падение программы.

## Как запустить тесты

Нужен `junit-platform-console-standalone-1.10.2.jar` (см. `KHARCHENKO_ITERATION0.md` — там ссылка на скачивание), положить в корень проекта:

```bash
javac -d out src/main/java/*.java
javac -cp "out:junit-platform-console-standalone-1.10.2.jar" -d out src/test/java/*.java
java -jar junit-platform-console-standalone-1.10.2.jar execute -cp out --scan-classpath
```

Ожидаемый результат: все тесты (`StepHistoryTest` + `GraphFileLoaderTest`, суммарно 12) — `successful`, `0 failed`.
